# _api/auth/password/
> L2 | 父级: /_api/auth/CLAUDE.md

## 成员清单

check.ts: 密码检查端点，验证密码是否正确，用于敏感操作二次确认
set.ts: 密码设置端点，修改用户密码，需要旧密码验证

## 依赖关系
- 依赖 _utils/auth 认证工具
- 依赖 _utils/_hash 密码哈希
- 依赖 Redis 存储密码哈希
- 被前端用户设置调用

## 密码管理约束
1. 修改密码必须验证旧密码
2. 新密码必须符合强度要求（至少 8 字符）
3. 密码哈希使用 bcrypt，salt rounds = 10
4. 修改密码后必须撤销所有旧令牌
5. 密码检查失败必须有速率限制
6. 密码不能与用户名相同
7. 密码修改成功后发送通知
